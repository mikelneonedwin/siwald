# Contributing to Siwald

We welcome contributions of new strategies and improvements to the tournament engine!

## Adding a New Player Strategy

To add a new player (e.g., a strategy generated by a new LLM model):

1. **Create a new file in `players/`**:
   Name it after your strategy, e.g., `players/my-new-model.ts`.

2. **Implement the `Player` interface**:
   Export a class that implements the `Player` interface from `types.d.ts`.

   ```typescript
   import { Player, Move, Level } from "../types";

   export class MyNewModel implements Player {
     level: Level = 1;
     xp: number = 100;

     makeChoice({ enemyLevel, xp, power }: { enemyLevel: Level; xp: number; power: number }): Move {
       // Your strategy logic here
       // enemyLevel: 1 or 2
       // xp: opponent's current XP (0-100)
       // power: your current turn's power (1-25)

       if (power > 20) return "attack";
       if (this.xp < 30) return "revive";
       return "50";
     }
   }
   ```

3. **Register your player**:
   Add your new class to the `PLAYERS` array in `players/index.ts`.

   ```typescript
   // players/index.ts
   import { MyNewModel } from "./my-new-model";

   export const PLAYERS: Array<new () => Player> = [
     // ... other players
     MyNewModel,
   ];
   ```

## Development Constraints

- **Single Method**: Your entire algorithm should ideally be in the `makeChoice` function.
- **No Extra Properties**: Do not add stateful properties to your class other than `level` and `xp`, as they are managed by the tournament engine.
- **Type Safety**: Ensure your implementation strictly follows the `Player` type definition.

## Running Tests

You can run the tournament to verify your player's performance:

```bash
bun run index.ts
```

Check the scoreboard in the console or open `dist/index.html` after the run.
